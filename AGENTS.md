# AGENTS.md

Этот файл задаёт bootstrap-entrypoint для агентных инструкций в репозитории.

## Базовое правило

При любом агентном запросе сначала прочитать:

- `.aiassistant/rules/00_COMMON.md`
- `.aiassistant/rules/specs/CONTROL_FLOW.md`

## Роутинг ролей

Если в сообщении пользователя есть явное упоминание роли в формате `@ROLE`, агент обязан дополнительно прочитать соответствующий role-файл и дальше работать строго в рамках этой роли.

Соответствие:

- `@LEAD` -> `.aiassistant/rules/LEAD.md`
- `@CODER` -> `.aiassistant/rules/CODER.md`
- `@TESTER` -> `.aiassistant/rules/TESTER.md`
- `@VERIFIER` -> `.aiassistant/rules/VERIFIER.md`
- `@REVIEWER` -> `.aiassistant/rules/REVIEWER.md`
- `@ANALYST` -> `.aiassistant/rules/ANALYST.md`

## Порядок применения

2. Затем, если указана роль через `@ROLE`, загрузить соответствующий role-файл.
3. Затем прочитать текущий запрос пользователя и активный `MAIL/**`-тикет.
4. Дочитывать только те артефакты, на которые активный тикет явно ссылается или без которых остаётся неустранимая неоднозначность.
5. Если упомянуто несколько ролей, запросить уточнение у пользователя и не выбирать роль по догадке.
6. Если файл роли отсутствует или недоступен, кратко сообщить об этом и продолжить с опорой на доступные инструкции.

## Интерпретация

- `@ROLE` является явным триггером переключения в соответствующую роль.
- Если `@ROLE` не указан, не принимать на себя роль по догадке, если это не следует напрямую из запроса пользователя.
- Этот файл является только router/bootstrap-слоем и не переопределяет `rules/specs`.

## Границы

- `.aiassistant/rules/*` остаются основным источником нормативных правил.
- `TEMPLATES/*` и `.aiassistant/skills/*` являются helper-layer и не могут переопределять `rules/specs` или активный тикет.
- Historical `MAIL/**` используется как evidence/history layer и не является bootstrap-слоем.
