# RIDER_DIFF_CONTEXT_COPY

## Назначение

Повторяемый recipe для задач, связанных с копированием diff-контекста в Rider-плагине.

## Базовые шаги

1. Уточнить ожидаемый формат копирования в активном тикете.
2. Проверить источники diff-данных и fallback-сценарии.
3. Реализовать/проверить поведение без выхода за текущий scope.
4. Зафиксировать сценарии проверки в отчёте.

## Ограничение

Skill помогает выполнять типовые шаги и не может переопределять `rules/specs`.
