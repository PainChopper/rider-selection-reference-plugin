# CONTROL_FLOW

Правила исполнения задач в этом репозитории.

## Базовый цикл

1. Прочитать входной артефакт задачи.
2. Собрать только необходимый контекст.
3. Выполнить задачу без выхода за scope.
4. Прогнать проверки, если они указаны или очевидно необходимы.
5. Обновить процессные артефакты, если задача этого требует.
6. Подготовить краткий отчёт с фактами, а не с предположениями.

## Работа со scope

- Не добавлять новые подзадачи без необходимости.
- Если обнаружена системная проблема, сначала исправить только то, что блокирует текущую задачу.
- Если проблема требует отдельного потока работ, зафиксировать это в отчёте, но не смешивать с текущей задачей.

## Runtime-policy для process-артефактов

- Для показа тикета или отчёта по умолчанию использовать печать содержимого в чате.
- `Start-Process` выполнять только при явном запросе пользователя или если это прямо требуется активным тикетом.
- Helper-layer (`TEMPLATES/*`, `.aiassistant/skills/*`) не может вводить альтернативную runtime-policy поверх этого раздела.

## Работа с процессными файлами

- Для `.windsurf/sandbox/MAIL/**` соблюдать формат имени, роль и ожидаемую директорию.
- Для `.windsurf/sandbox/00_STATE.md` отражать только фактические изменения процесса.
- Не переписывать исторические записи без отдельной причины.
- Печатать process-файлы в чате нужно обычным Markdown-текстом, без fenced code block.

## Ownership и update-flow для `00_STATE.md`

- LEAD отвечает за синхронизацию `.windsurf/sandbox/00_STATE.md` по факту.
- После завершения тикета соответствующие `IN`/`OUT`-артефакты переводятся в `Done`; `Active` должен отражать фактическое текущее состояние.
- Для созданных/обновлённых process-артефактов текущего тикета обязательна постановка в git index до завершения работы.

## Проверки

- Перед завершением убедиться, что изменённые пути существуют и ссылки на них корректны.
- Если команда или артефакт обязательны по инструкции, явно отметить факт их выполнения.
- Перед завершением убедиться, что не создан новый bootstrap-cycle между `AGENTS.md` и `rules/specs`.
