# 00_COMMON

Общие правила для всех ролей в репозитории `D:\!_DEV_!\JetBrainsPlagins`.

## Цель репозитория

- В этом репозитории ведётся работа над отдельными JetBrains-плагинами, начиная с Rider-плагина.
- Все изменения выполняются только внутри `D:\!_DEV_!\JetBrainsPlagins`.
- Не изменять файлы вне этого каталога, если это не запрошено отдельно и явно.

## Обязательный порядок

1. Сначала прочитать `AGENTS.md`.
2. Затем прочитать этот файл.
3. Затем прочитать `.aiassistant/rules/specs/CONTROL_FLOW.md`.
4. Затем прочитать `.aiassistant/rules/specs/FRONT_BACKEND_CONTRACT.md`.
5. Если указана роль `@ROLE`, прочитать соответствующий role-файл и работать в его рамках.

## Общие ограничения

- Не расширять scope задачи без явного запроса.
- Сначала опираться на локальные файлы репозитория и sandbox-артефакты.
- Если в инструкции упомянуты отсутствующие файлы, кратко зафиксировать это и продолжить по доступному контексту.
- Для правок существующих файлов сохранять текущий процессный стиль и структуру.
- Для новых process/mail/state-файлов использовать уже принятые в `.windsurf/sandbox` соглашения по именованию.

## Ожидания к результату

- Изменения должны быть минимально достаточными.
- Отчёт должен явно перечислять, что изменено и что проверено.
- Если меняются `MAIL`-файлы, соблюдать требования текущего процесса для этих файлов.
- Если агент сформировал или обновил process-файл, он должен напечатать содержимое этого файла прямо в финальном отчёте пользователю.
- При печати process-файла в финальном отчёте не использовать code block; печатать нужно как обычный Markdown-документ.
