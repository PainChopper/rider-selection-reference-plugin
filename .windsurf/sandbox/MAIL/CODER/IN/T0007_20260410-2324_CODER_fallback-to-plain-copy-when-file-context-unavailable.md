## Контекст

После внедрения LLM-формата (`[FILE]`, `[LINES]`, `[LANG]`, `[CODE]`) подтверждено, что в файловом контексте это работает отлично.

Нужно скорректировать поведение для нефайлового/неопределимого контекста (например, консоль, служебная вкладка, случай, когда путь нельзя надёжно определить): вместо полного `return` action должен копировать просто выделенный текст, как обычный copy (`Ctrl+C` / `Ctrl+Insert`), без служебной обвязки.

## Scope

- Изменить поведение `Copy Selection with Path and Lines` только для кейсов, где файловый контекст недоступен/невалиден.
- В таких кейсах класть в буфер только текст выделения.
- Для валидного файла внутри проекта сохранить текущий LLM-формат без изменений.
- Не менять хоткей, UI, настройки и прочее.

## Целевое поведение

1. Если есть непустое выделение и корректный файловый контекст внутри проекта:
   - оставить текущий формат:
     - `[FILE] <path>`
     - `[LINES] <lineStart>-<lineEnd>`
     - `[LANG] <language-or-text>`
     - `[CODE]`
     - `<selectedTextNormalized>`
     - `[/CODE]`
2. Если есть непустое выделение, но файловый контекст определить нельзя:
   - копировать в буфер только текст выделения (plain text), без `[FILE]/[LINES]/[LANG]/[CODE]`.
3. Если выделение пустое:
   - поведение без изменений (ничего не копировать).

## Какие кейсы считать "файловый контекст определить нельзя"

- `VIRTUAL_FILE` отсутствует.
- файл не в local file system.
- путь файла не парсится/невалиден.
- путь проекта невалиден или отсутствует.
- файл не принадлежит текущему проекту.
- относительный путь пустой.

Для этих кейсов: fallback на plain copy выделенного текста.

## Технические ограничения
