Использовать этот skill для задач around diff-context copy в Rider-плагине.

## Что делать

1. Зафиксировать ожидаемый copy-format и scope активного тикета.
2. Найти реальный источник diff-данных и проверить fallback-сценарии.
3. Реализовать или проверить изменение без выхода за текущий scope.
4. Явно перечислить сценарии проверки и ограничения в отчёте.


Этот skill не переопределяет Owner instruction, rules/specs или active ticket.
