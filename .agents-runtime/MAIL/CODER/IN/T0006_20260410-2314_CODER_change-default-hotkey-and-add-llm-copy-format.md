## Контекст

Текущий Rider-плагин уже работает в живом сценарии и проходит минимальные проверки. После ручного использования выявлены два улучшения следующего порядка:

1. Дефолтный хоткей action конфликтует со встроенной функциональностью Rider, поэтому его нужно изменить.
2. Текущий формат копирования хорошо читается человеком, но не оптимален для передачи фрагментов кода в LLM. Нужен структурированный, максимально нативный для модели формат, чтобы путь, диапазон строк, язык и сам код были явно отделены от сопроводительного текста.

Пользователь уже выбрал новый дефолтный shortcut: `Ctrl+Alt+Shift+Z`.

## Scope

- Изменить дефолтный shortcut action `Copy Selection with Path and Lines` на `Ctrl+Alt+Shift+Z`.
- Обновить формат текста, который action кладёт в буфер обмена.
- Новый формат должен быть ориентирован на LLM и быть максимально однозначным по структуре.
- Сохранить вычисление относительного `path`, диапазона строк и нормализацию перевода строк.
- Сохранить все существующие защитные условия: пустое выделение, нефайловые/служебные вкладки, несохранённые документы и файлы вне проекта по-прежнему не должны менять буфер.

## Новый целевой формат буфера обмена

Формат должен быть ровно таким:

`[FILE] <path>`
`[LINES] <lineStart>-<lineEnd>`
`[LANG] <language-or-text>`
`[CODE]`
`<selectedTextNormalized>`
`[/CODE]`

Требования к полям:
- `<path>` — относительный путь от текущего проекта с `/`.
- `<lineStart>-<lineEnd>` — текущая логика диапазона без изменений.
- `<language-or-text>` — определить по расширению файла, если это можно сделать минимально и надёжно; если нет надёжного определения, использовать `text`.
- Между строками не добавлять лишних префиксов, суффиксов и пустых строк.
- Маркеры `[CODE]` и `[/CODE]` обязательны.

## Не делать

- Не менять общую функциональность action кроме хоткея и формата буфера.
- Не добавлять новые настройки UI, диалоги, конфиги или экран параметров.
- Не усложнять определение языка тяжёлой логикой, если достаточно минимально достаточного отображения extension -> language.
- Не менять файлы вне `D:\.DEV\JetBrainsPlagins`.
